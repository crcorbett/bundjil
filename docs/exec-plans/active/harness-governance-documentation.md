# Bundjil harness-governance documentation rollout

This plan owns Bundjil's current bounded sequence. The cross-repository ledger
owns aggregate dependencies; the repository-local SPEC owns Bundjil outcomes
and task state.

## Progress

- **HGI-300 — in progress:** establish one lifecycle router, one current
  architecture route, current SPEC/plan ownership, truthful active/history
  routes and historical-ledger field classification, root README
  decontamination, and accounting receipts.
- HGI-301, HGI-302, HGI-307, HGI-303, HGI-304, HGI-305, and HGI-306 remain
  pending in dependency order.

HGI-300-S1 preserves historical ledgers (including their distinct task-array,
ledger-lifecycle, and recorded-approval fields), completed plans, research,
and the old mixed architecture atlas. It stops before package README/runbook
extraction, provider access, deployment, evidence deletion/movement, or another
unresolved semantic-owner class.

Acceptance requires corpus/link receipts, held-out retrieval,
`bun run check:skills`, `bun run verification`, and a scoped Git receipt. Local
checks do not prove provider/runtime/production state.
